How Vector Search Works

                
            


            

                When you searched for "", here's what happened behind the scenes:
            


            

                
1

                
Query Embedding

                
Your search phrase was converted into a 1,536-dimensional vector using OpenAI's text-embedding-3-small model. Each dimension represents a learned semantic feature that captures the meaning of your query.

                

            


            

                
2

                
Property Embeddings

                
Every property description in the database was previously converted to a 1,536-dimensional vector using the same model. This allows SQL Server to perform meaningful mathematical comparisons between your query and all properties.

            


            

                
3

                
Cosine Distance Calculation

                
SQL Server 2025 uses the VECTOR_DISTANCE('cosine', ...) function to calculate the angular distance between your search vector and each property vector:

                
    
                    
  • 0.0 = Identical vectors (perfect semantic match)
    • 
                    
  • 0.1-0.3 = Very similar (typical range for good matches)
    • 
                    
  • 0.5-1.0 = Somewhat related
    • 
                    
  • 1.0-2.0 = Increasingly different
    • 
                    
  • 2.0 = Completely opposite vectors
    • 
                

            


            

                
4

                
Result Ranking

                
Properties are sorted by distance in ascending order. Lower distances appear first because they're more semantically similar to your query. This is how semantic search works‚Äîit understands meaning, not just keywords.
